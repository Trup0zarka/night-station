# CitiNet Console Migration Plan (Parallel UI Transition)

Этот документ описывает процедуру создания новых интерфейсов в стилистике **CitiNet** параллельно существующим (Legacy) консолям. Это обеспечивает плавный переход, позволяя использовать обе версии интерфейса одновременно.

## 1. Исследование (Research)
*   **Идентификация:** Выбрать консоль для миграции (например, `NcpdForensics`).
*   **Анализ BUI:** Изучить существующие ключи интерфейса (`UiKey`) и сообщения (`Messages`) в `Shared` проекте.

## 2. Дублирование и Расширение (Shared)
*   **Новый UiKey:** Добавить новый ключ интерфейса специально для CitiNet версии (например, `NcpdForensicsUiKey.CitiNet`).
*   **Сообщения:** Убедиться, что существующие сетевые сообщения подходят для обоих интерфейсов.

## 3. Создание Нового Интерфейса (Client)
*   **Новый Контрол:** Создать новый XAML-контрол, визуально интегрированный в стиль CitiNet.
*   **Новый BUI:** Создать новый класс `BoundUserInterface` (например, `NcpdForensicsCitiNetBui.cs`), который:
    *   Использует новый `UiKey.CitiNet`.
    *   Открывает стилизованное окно-обертку CitiNet.
    *   Взаимодействует с тем же серверным состоянием, что и оригинальный BUI.

## 4. Поддержка на Сервере (Server)
*   **Регистрация BUI:** Обновить серверную систему, чтобы она подписывалась на сообщения и отправляла обновления состояния на ОБА ключа интерфейса (`Legacy` и `CitiNet`).
*   **Унификация логики:** Логика обработки данных остается общей, гарантируя идентичное поведение обеих версий.

## 5. Прототипирование (YAML)
*   **Legacy Entity:** Оставить существующий прототип консоли без изменений.
*   **CitiNet Entity:** Создать новый прототип (например, `id: NcpdForensicsConsoleCitiNet`), который:
    *   Использует новый `UiKey.CitiNet`.
    *   Имеет обновленный спрайт или экран в стиле CitiNet.
    *   Сохраняет все оригинальные компоненты логики.

## 6. Тестирование и Валидация
*   **Синхронизация:** Действия, совершенные в CitiNet-интерфейсе, корректно отображаются в Legacy-интерфейсе и наоборот.
*   **Стабильность:** Убедиться, что наличие двух открытых интерфейсов к одной сущности не вызывает конфликтов.
*   **Доступ:** Проверить, что права доступа (ID-карты) корректно применяются к обоим типам консолей.
