# Дизайн-документ: Экосистема CitiNet и Модульные Терминалы

## 1. Общее описание
Замена стандартных консолей SS14 на аутентичную систему настольных ПК и портативных кибердек, работающих через единую сеть **CitiNet**. Вместо разрозненных интерфейсов используется концепция "Сайтов" и "Приложений", доступ к которым управляется правами пользователя и физическими носителями данных (Data Chips).

## 2. Аппаратное обеспечение (Hardware)

### 2.1. Настольный ПК (Desktop Terminal)
*   **Визуал:** Громоздкий корпус, ЭЛТ-монитор с эффектом мерцания, внешняя клавиатура.
*   **Тип:** Стационарная сущность.
*   **Особенности:** 
    *   Высокая скорость передачи данных.
    *   Требует прямого подключения к электросети.
    *   Слот для одного Data Chip.

### 2.2. Кибердека (Cyberdeck)
*   **Визуал:** Плоский блок с разъемами, носится в руках или на поясе.
*   **Тип:** Портативный предмет.
*   **Особенности:**
    *   Позволяет открывать интерфейс CitiNet в любом месте.
    *   Возможность подключения к внешним портам для взлома.
    *   Ограниченное время работы (аккумулятор).

### 2.3. Чипы данных (Data Chips)
*   **Назначение:** Расширение функционала ПК.
*   **Механика:** При установке чипа в слот ПК, в браузере открывается новый "Сайт" (например, зашифрованный рынок или скрытая база данных).

## 3. Программная архитектура (CitiNet)

### 3.1. Система Сайтов (Net-Sites)
Вместо жестко закодированных консолей (Medical Console, Security Console), весь функционал переносится на прототипы "Сайтов".
*   **Структура прототипа:**
    *   `ID`: Уникальный идентификатор сайта.
    *   `Name`: Название (например, "NCPD Central Database").
    *   `URL`: Визуальный адрес (например, `ncpd.gov/records`).
    *   `Icon`: Иконка для панели быстрого доступа.
    *   `RequiredAccess`: Список прав (Tags) ID-карты для открытия.
    *   `BUI`: Ссылка на класс интерфейса (BoundUserInterface).

### 3.2. Браузер (Main Interface)
Универсальный UI, который отрисовывается при взаимодействии с любым компьютером.
*   **Header:** Строка адреса, статус подключения, имя текущего пользователя (из ID-карты).
*   **Sidebar:** Список "закладок" — доступных пользователю сайтов.
*   **Viewport:** Область, в которой отображается содержимое выбранного сайта.

## 4. Доступ и Безопасность

### 4.1. Авторизация
*   Используется существующая система `AccessReader`.
*   При взаимодействии с ПК происходит автоматическая попытка логина по ID-карте в слоте или в руках пользователя.
*   Список доступных сайтов в Sidebar динамически обновляется в зависимости от прав доступа карты.

### 4.2. Теневой доступ (Аплинки)
*   "Аплинки" предателей теперь являются скрытыми сайтами в CitiNet.
*   **Способы получения доступа:**
    1.  **Чип доступа:** Покупается у Фиксера или находится в тайнике. Разблокирует сайт при вставке чипа в ПК.
    2.  **Секретный URL:** Знание конкретного адреса, который нужно ввести вручную.

## 5. Техническая реализация (ECS)

### 5.1. Компоненты
- `NetBrowserComponent`: Хранит текущую сессию, открытый сайт и список "кэшированных" (разблокированных) адресов.
- `DataChipComponent`: Содержит данные о том, какой сайт или программу он разблокирует.
- `NetSitePrototype`: Описание метаданных сайта в YAML.

### 5.2. Системы
- `NetBrowserSystem`: Обрабатывает запросы на открытие сайтов, проверяет права доступа через `AccessReaderSystem`.
- `DataChipSystem`: Логика вставки/извлечения чипов и обновления списка доступных сайтов.

## 6. Примеры контента (Initial Sites)
1.  **Public:** `nightcity.gov` — новости, законы, общая информация.
2.  **Security:** `ncpd.net/database` — база данных преступников, управление тюрьмой.
3.  **Medical:** `traumacare.com/monitor` — мониторинг состояния клиентов Trauma Team.
4.  **Night Market:** `night-market.onion` — (Аплинк) покупка нелегального софта и оружия.
